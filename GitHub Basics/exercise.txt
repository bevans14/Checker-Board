1.	The terminal is a way to directly access and modify any files, commands, data, and programs to the console of a computer. 
2.	Git is a version control system that makes accessing and viewing any changes made to code and files easier, while Github is a public website that allows someone to store those files and changes.
3.	3. To fork a repository, go to the page of that repository, click on the ‘’fork” button, choose where you want to fork that repository, and wait for Github to fork it. Feel free to then make any changes to that fork as needed.
